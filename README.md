# Fidelity.Image

Pure-computation image decoding and manipulation library for the Fidelity ecosystem.

Fidelity.Image is a Tier 2 library that provides platform-specific image decoding backends behind a common API surface. It operates headlessly with no display dependencies, making it suitable for both UI rendering pipelines and batch image processing.

## Architecture

Fidelity.Image sits between platform bindings (Tier 1) and application code (Tier 3):

```
Application (HelloWayland, Warhol, ...)
        |
  Fidelity.Image          ← Tier 2: idiomatic decode/query API
        |
  Platform backends        ← Tier 1: Farscape-generated FidelityExtern bindings
    ├── Linux: stb_image
    ├── macOS: ImageIO/Accelerate (future)
    └── Windows: WIC (future)
```

## Current Status

**Linux x86-64** — stb_image backend providing:
- PNG, JPEG, BMP, GIF, TGA, HDR, PSD, PIC, PNM decode
- 8-bit, 16-bit, and float-per-channel loading
- Image info queries without full decode
- HDR detection and format queries
- Thread-safe error reporting via `NullWithReason` convention

## Platform Bindings

Bindings are generated by [Farscape](https://github.com/SpeakEZ/Farscape) from the pilot TOML in `CPU/Linux/x86_64/pilot/stb_image.pilot.toml`.

**Layer 1** — `[<FidelityExtern>]` declarations mapping directly to stb_image C functions.

**Layer 2** — Idiomatic wrappers with `Result<nativeint, nativeint>` return types. Functions that return allocated pointers (all `stbi_load*` variants) automatically call `stbi_failure_reason()` on null and wrap the error in `Result`.

## Vendor Library

`CPU/Linux/x86_64/vendor/` contains the build artifacts for stb_image:
- `stb_image_impl.c` — Compilation unit that defines `STB_IMAGE_IMPLEMENTATION`
- `libstb_image.a` — Pre-compiled static library (link target)

stb_image is public domain software (Unlicense / MIT dual license) from [nothings/stb](https://github.com/nothings/stb).
